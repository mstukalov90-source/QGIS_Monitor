# Monitor DB Loader

QGIS-плагин для подключения к PostgreSQL (БД Monitor) и загрузки пространственных слоёв с символикой из JSON-конфигурации.

## Целевая версия

**QGIS 3.44 LTR** (рабочая среда, Qt5 / PyQt5).

Плагин также устанавливается на **QGIS 4.x** (Qt6) — enum Qt вынесены в `core/qt_compat.py`.

| Версия QGIS | Каталог профиля (плагины) |
|-------------|---------------------------|
| 3.44 LTR | `QGIS3/profiles/default/python/plugins/` |
| 4.x | `QGIS4/profiles/default/python/plugins/` |

### Пути к каталогу плагинов

**Windows (рабочий ПК):**

```text
%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\
```

**Linux:**

```text
~/.local/share/QGIS/QGIS3/profiles/default/python/plugins/
```

**macOS:**

```text
~/Library/Application Support/QGIS/QGIS3/profiles/default/python/plugins/
```

## Требования

- QGIS **3.44** или новее (до 4.x включительно)
- Доступ к PostgreSQL/PostGIS (параметры в `monitor_db_loader/resources/layers_config.json`)
- Рекомендуется **psycopg2** в Python QGIS (автоопределение геометрического столбца)

Проверка psycopg2 в консоли Python QGIS:

```python
import psycopg2
```

Если модуля нет — установите через OSGeo4W / pip в окружение QGIS (на Windows часто: `python -m pip install psycopg2-binary` из OSGeo4W Shell).

## Установка

### Вариант A: из ZIP (рекомендуется)

1. Соберите архив из корня репозитория:

   ```bash
   ./scripts/build_plugin_zip.sh
   ```

   Файл: `dist/monitor_db_loader.zip`

2. В QGIS 3.44: **Модули → Управление модулями → Установить модуль из ZIP…**
3. Выберите ZIP, включите **Monitor DB Loader** на вкладке «Установленные».

### Вариант B: копирование каталога

Скопируйте папку `monitor_db_loader` в каталог плагинов (см. таблицу выше) и перезапустите QGIS.

## Использование

- Загрузка слоёв — по кнопке **Загрузить слои Monitor DB** в меню **Monitor DB Loader** или на панели инструментов.
- **Первичный анализ фото** — вторая кнопка того же меню; работает по уже загруженным слоям проекта (пароль БД не запрашивается).
- Автозапуск при старте QGIS отключён (`load_on_startup: false` в конфиге); при необходимости можно включить, установив `load_on_startup: true` в `layers_config.json`.
- Пароль не сохраняется.
- Ошибка подключения — сообщение и **Повторить**.
- Ошибка слоя — пропуск, запись в журнал **Monitor DB Loader**.

### Первичный анализ фото

После загрузки слоёв кнопка **Первичный анализ фото** открывает диалог выбора **района** из слоя «Границы районов» (поле `rayon`, поиск по списку). Анализ выполняется только для выбранного полигона:

- **Исходные точки** — только внутри границы района
- **Опорные объекты** — пересекающие полигон района (корректно у границ)

Это ускоряет расчёт на больших объёмах данных. Слой «Границы районов» должен быть загружен в проект (находится в группе «Справочная информация», по умолчанию скрыта).

#### Исходные слои (анализируемые точки)

| Слой |
|------|
| Фотографии после обработки ИИ |
| Фото разрытий и строек |
| Разрытия из паспортов ОГХ |

Классифицируются точки из перечисленных слоёв по близости к опорным объектам. Результат — группа **Первичный анализ фото — {район}** (вставляется первой в дереве слоёв):

| Слой | Условие | Цвет |
|------|---------|------|
| **Зелёная таблица** | ≤ 25 м от любого объекта **Группы А** | зелёный |
| **Жёлтая таблица** | ≤ 50 м от **Группы Б**, если точка не попала в зелёную | жёлтый |
| **Красная таблица** | все остальные точки внутри района | красный |

#### Группа А (радиус 25 м)

| Слой / группа | Подслои |
|---------------|---------|
| Разрытия из паспортов ОГХ | точки |
| Ордера ОАТИ | точки, линии, полигоны |
| Данные о действующих зарегистрированных уведомлениях на проведение земляных работ | точки, линии, полигоны |
| Действующие уведомления на аварийно-восстановительные работы и текущие локальные ремонты | точки, линии, полигоны |

#### Группа Б (радиус 50 м)

Все слои **Группы А**, плюс:

| Слой / группа | Подслои |
|---------------|---------|
| Полигоны СтройМониторинг | полигоны |
| Объекты капитального строительства | точки, полигоны |

Каждая исходная точка попадает **строго в одну** таблицу (приоритет: зелёная → жёлтая → красная).

Для точек зелёной и жёлтой таблиц в атрибутах сохраняются ссылки на все опорные объекты, по которым точка попала в класс:

- `ref_layers` — имена слоёв (через `; `)
- `ref_fids` — идентификаторы объектов
- `ref_distances_m` — расстояния в метрах

На карте рисуются пунктирные линии связей:

- **Зелёные связи** — от зелёных точек к каждому объекту Группы А
- **Жёлтые связи** — от жёлтых точек к каждому объекту Группы Б

Повторный запуск анализа для того же района заменяет предыдущую группу с тем же именем. Параметры (радиусы, слои, район, цвета) задаются в секции `photo_primary_analysis` файла `layers_config.json` (подсекция `district_filter`).

## Конфигурация

`monitor_db_loader/resources/layers_config.json` — подключение, группы слоёв, символика.

У каждого слоя в JSON указаны **`schema`** и **`geometry_column`**. Актуальная структура БД:

| Схема | Слои | Geom-столбец |
|-------|------|----------------|
| `data_mos` | `items_*`, `items_*_points`, `items_*_lines`, `items_*_polygons` | `geom` |
| `vector_stroy` | `url_222` | `geom` |
| `stroymonitoring` | `boundaries_aip` | `geom_valid` |
| `lens` | `reports`, `contours*` | `geom` / `geometry` |
| `genplan` | `photo_meta`, `order` | `geom` |
| `odh_export` | `hood`, `dgi`, `oozt`, `ogh-disruption` | `geom` / `geometry` |

Смешанная геометрия для ордеров и уведомлений берётся из отдельных таблиц `items_{id}_points`, `_lines`, `_polygons` (без SQL-фильтра по `ST_GeometryType`).

### Дерево групп (верхний уровень)

1. **Данные МГГТ** — ГенПлан, «Объектив», MicroStation (разрывия ОГХ, заглушки КГС/АВР/ОПС), МосСтройМониторинг
2. **Данные полученые от г. Москва** — ордера ОАТИ, капстроительство, уведомления земляных работ, АВР/локальные ремонты
3. **Справочная информация** — камеры ЕЦХД, границы районов, объекты ДГИ и ООЗТ (по умолчанию скрыта)

При загрузке все группы свёрнуты (`collapse_groups_on_load: true`). Пустые слои-заглушки задаются флагом `"placeholder": true` в JSON.

В конфиге поддерживается вложенность: у группы могут быть `layers` и вложенные `groups`.

Секция **`photo_primary_analysis`** — параметры первичного анализа фото: исходные и опорные слои/группы, радиусы (25 м / 50 м), фильтр по району (`district_filter`), имена и цвета результирующих таблиц и слоёв связей.

## Ручная проверка (QGIS 3.44)

1. Установить из ZIP, включить плагин.
2. Диалог пароля при старте; неверный пароль → «Повторить».
3. В дереве слоёв: 3 корневые группы, все свёрнуты; вложенная структура по разделу «Дерево групп» выше.
4. GenPlan — только `disruption = true`; **Разрытия из паспортов ОГХ** — чёрные пирамиды; КГС/АВР/ОПС — пустые заглушки; группа **Справочная информация** и **Объекты ДГИ и ООЗТ** скрыты по умолчанию.
5. Повторная загрузка удаляет прежние корневые группы плагина.
6. Частичная загрузка при недоступной БД — счётчик «N из M» и лог.
7. **Первичный анализ фото**: диалог выбора района → группа «Первичный анализ фото — {район}» с тремя таблицами и слоями связей; точки вне района не учитываются; повторный запуск для того же района заменяет результаты.

## Безопасность

Пароль не пишется в JSON, `QSettings` и логи. Параметры хоста/пользователя — в JSON внутри плагина.

## Где смотреть логи, если слои не загрузились

### 1. Журнал сообщений QGIS (главное)

**Вид → Панели → Журнал сообщений** (англ. *View → Panels → Log Messages*).

На вкладке **Monitor DB Loader** — все сообщения плагина: подключение, каждая таблица, ошибки geom и PostGIS.

На вкладке **PostGIS** / **General** — ошибки драйвера PostgreSQL (сеть, SSL, права).

Включите уровни **Info** и **Warning** (иконки фильтра в панели), иначе часть записей может быть скрыта.

### 2. Файл лога на диске

| ОС | Путь (профиль по умолчанию) |
|----|----------------------------|
| Windows | `%APPDATA%\QGIS\QGIS3\profiles\default\qgis\qgis.log` |
| macOS | `~/Library/Application Support/QGIS/QGIS3/profiles/default/qgis/qgis.log` |
| Linux | `~/.local/share/QGIS/QGIS3/profiles/default/qgis/qgis.log` |

Ищите строки с `Monitor DB Loader` или ошибки `postgres` / `PostGIS`.

### 3. Консоль Python (для разовой отладки)

**Модули → Консоль Python** — при ручном запуске плагина смотрите traceback, если окно «зависло» без сообщения.

### Типичные причины «долго думал, 0 слоёв»

- Нет доступа к хосту `77.222.63.161:5432` с рабочего ПК (VPN, файрвол).
- Неверная схема/таблица или столбец геометрии не `geom` — в логе: «Не найден геометрический столбец».
- Без psycopg2 перебор 18×4 подключений к БД занимает много времени.
- Проверка подключения прошла, но таблицы в другой схеме, не `public`.

После обновления плагина (v1.0.2+) журнал **Monitor DB Loader** показывает пошаговую загрузку каждого слоя.
